## ok

```sh
To test a specific question, use the -q (or --question) option with the name of
the question:

    python3 ok -q foo
    python3 ok -q 12

By default, only tests that fail will appear. If you want to see the results
from all tests, you can use the -v (or --verbose) option:

    python3 ok -q foo -v

To start an interactive interpreter after a failed test for debugging, use the
-i (or --interactive) option:

    python3 ok -q foo -i

By default, after each test run ok will attempt to back up your work to the
server. To run the tests without any network access, use the --local option:

    python3 ok -q foo --local

To submit the assignment after you're done, use the --submit option:

    python3 ok --submit

Finally, to log out and log in under a different email, use --authenticate:

    python3 ok --authenticate

Visit https://okpy.org to view your backups and submissions.

options:
  -h, --help            show this help message and exit

running tests:
  -q, --question QUESTION
                        run tests for a specific question
  --suite SUITE         run cases from a specific suite
  --case CASE           run specific cases
  -u, --unlock          unlock tests interactively
  -i, --interactive     start the Python interpreter after a failed test
  -v, --verbose         show all tests (not just passing tests) up to failing line (if
                        any)
  -t, --testing [TESTING]
                        run tests from rst file (default: mytests.rst)
  --all                 run tests for all questions in config file
  --submit              submit the assignment
  --backup              attempt to reliably backup your work
  --revise              submit composition revision
  --timeout TIMEOUT     set the timeout duration (in seconds) for running tests
  -cov, --coverage      get suggestions on what lines to add tests for
  --autobackup          back up your work every minute in the background

debugging tools for students:
  --trace               trace code and launch python tutor
  --trace-print         print the trace instead of visualizing it

experiment options:
  --no-experiments      do not run experimental features
  --hint                give a hint (if available)
  --style               run AutoStyle feedback system
  --collab              launch collaborative programming environment
  --get-help            receive 61A-bot feedback on your code
  --consent             get 61A-bot research consent

ok developer debugging options:
  --version             print the version number and exit
  --tests               display a list of all available tests
  --debug               show debugging output

grading options:
  --lock                lock the tests in a directory
  --score               score the assignment
  --score-out [SCORE_OUT]
                        write scores to a file
  --config CONFIG       use a specific configuration file
  --ignore-empty        ignore empty doctests

encryption:
  --generate-encryption-key GENERATE_ENCRYPTION_KEY
                        generates a JSON file containing a list of [(file, key)] pairs.
                        Path is a keyfile
  --encrypt ENCRYPT     encrypt each problem. provide a path to a keyfile generated by
                        --generate-encryption-key
  --encrypt-padding ENCRYPT_PADDING
                        If provided, pads all plaintexts to this size in bytes.Errors if
                        any of the files are longer than this
  --decrypt [DECRYPT ...]
                        decrypt all problems where the given keys apply

server options:
  --local               disable any network activity
  --nointeract          disable prompts to user
  --server SERVER       set the server address
  --authenticate        authenticate, ignoring previous authentication
  --no-browser          do not use a web browser for authentication
  --get-token           get ok access token
  --insecure            use http instead of https
  --no-update           do not check for ok updates
  --update              update ok and exit
  --parsons             run parsons problems in browser
```
